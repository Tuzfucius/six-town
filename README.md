# 杭湖嘉绍六镇新质生产力数字图谱平台

面向“杭湖嘉绍都市圈小镇经济新质生产力调研”的沉浸式数字图谱前端。平台以六个特色小镇为入口，将产业定位、代表企业、空间点位与调研叙事组织为可浏览的信息页面和三维地图页面。

## 项目内容

覆盖梦想小镇、人工智能小镇/未来科技城、中国（杭州）算力小镇、德清地理信息小镇、越城集成电路小镇和嘉善归谷智造小镇。首页负责六镇总览与入口分发；每个小镇包含产业叙事、企业摘要、产业分类与空间标记；地图页提供以 MapLibre 为底图的空间浏览界面。

## 架构概览

```text
src/main.tsx
  └─ src/App.tsx
      ├─ /                 → LandingView
      ├─ /:townId/info     → TownView
      └─ /:townId/map      → TownMapView
          └─ townsData（六镇统一内容模型）
```

`src/data/towns.ts` 是当前单一内容源，按小镇 ID 存放名称、坐标、统计指标、企业摘要、图谱分类、地图标记和展示文案。`TownView` 负责内容展示与详情叙事，`TownMapView` 使用 MapLibre 依据 `mapCoords` 与 `mapData.markers` 渲染地图，`TownBackground` 按小镇类型提供不同的动态背景。

## 内容与开发约束

- 企业、地址、产品、项目状态与小镇归属必须区分“已核验”“部分核验”“待核验”；不可将候选线索写成已证实事实。
- 地址数据应区分注册地址、实际办公地、研发地、生产基地与历史孵化地；地图点位仅使用已确认或明确标注精度与状态的数据。
- 六镇产业统计使用统一的一级标签、二级标签和产业链位置字段；企业可有多个标签，但主标签与产业链环节不得混用。
- 当前数据为前端展示样本，不得表述为园区全部企业或完整空间分布。对外发布前应完成政府/管委会、企业官网或原始材料的交叉核验。
- 组件应继续复用 `townsData` 的统一模型；新增小镇或企业时，避免将业务数据硬编码到视图组件。
- 页面使用浏览器路由，部署到静态托管环境时必须配置 SPA 回退到 `index.html`。
- 地图样式当前依赖 Carto 的在线深色底图；离线发布或政务内网部署需替换为合规的可用底图服务。

## 1. 项目概述

### 1.1 项目背景

本平台服务于“杭湖嘉绍都市圈小镇经济新质生产力调研”，以六个特色小镇为对象，将公开地图、企业资料、产业信息、交通区位和调研成果整合为一个可交互的数字地图产品。

平台不定位为专业测绘系统、政务监管平台或企业工商数据库，而是定位为：

> **以三维地图为主体的六镇数字展馆、产业导览与空间叙事平台。**

用户进入平台后，应当能够完成以下行为：

- 从都市圈尺度认识六个小镇的空间关系；
- 进入某个小镇，以三维园区视角浏览空间结构；
- 通过企业“任务点”查看代表企业、技术成果与产业定位；
- 切换产业热力、交通可达和六镇比较等分析模式；
- 理解“软—融—算—空—芯—智”六类新质生产力路径；
- 获取企业信息、资料来源和调研说明。

### 1.2 核心产品价值

平台需要同时满足三类价值：

1. **展示价值**  
   用于社会实践答辩、成果汇报、管委会交流、公众传播。

2. **认知价值**  
   帮助用户快速理解六镇的产业定位、企业分布和空间关系。

3. **复用价值**  
   页面截图、企业卡片、产业图层和比较结果可直接用于报告、海报、视频与PPT。

## 2. 产品定位与边界

### 2.1 产品定位

平台的产品表达应接近以下组合：

- 数字展馆；
- 三维园区导览；
- 互动地图叙事；
- 轻量产业分析工具；
- 研究成果可视化入口。

### 2.2 明确不做的内容

第一阶段不建设以下能力：

- 专业GIS编辑与制图；
- 在线修改建筑或道路数据；
- 高精度测绘与厘米级定位；
- 完整工商征信系统；
- 实时交通与实时人口监控；
- 复杂用户权限与账号体系；
- 六镇全部建筑的精细三维建模；
- 未经验证的产业评分和排名；
- 将全部公开企业一次性纳入平台。

### 2.3 设计目标

平台应实现：

- 全屏沉浸式体验；
- 地图始终为视觉主体；
- 信息按交互逐步出现；
- 多功能但不堆叠；
- 视觉现代、克制、清晰；
- 桌面端答辩演示稳定；
- 移动端可浏览、可操作；
- 数据来源和数据边界明确。

## 4. 总体设计原则

### 4.1 地图优先

地图占据主要视觉面积。导航、筛选、详情和分析控件均以悬浮胶囊、浮层、抽屉或底部面板出现。

### 4.2 渐进式披露

默认只展示最重要的信息：

- 六镇位置；
- 当前小镇；
- 重点企业；
- 核心操作。

详细企业信息、图层设置、数据口径和分析结果均在用户主动操作后展开。

### 4.3 单一主任务

每个界面状态只突出一个主要任务：

- 首页：进入地图；
- 总览：选择小镇；
- 探索：浏览园区；
- 企业：查找企业；
- 产业：查看分布；
- 交通：理解区位；
- 比较：识别差异。

### 4.4 视觉克制

避免：

- 传统政务大屏；
- 大量矩形统计卡片；
- 蓝色霓虹边框；
- 持续闪烁的点位；
- 满屏滚动数字；
- 复杂仪表盘；
- 同时显示所有图层。

### 4.5 数据诚实

对以下内容必须标注数据口径：

- 企业数量；
- 热力图；
- 建筑高度；
- 通勤圈；
- 企业实际地址；
- 产业分类；
- 公开信息更新时间。

## 7. 动效系统

### 7.1 动效目标

动效用于：

- 建立空间连续性；
- 表达页面层级；
- 引导用户注意；
- 解释状态变化。

不用于装饰性炫技。

### 7.2 核心动效

#### 首页加载

- 背景图轻微缩放退出；
- 标题模糊上升；
- 副标题延迟淡入；
- 导航与按钮依次出现。

#### 地图切换

- 选择小镇后执行平滑 `flyTo`；
- 飞行期间弱化其他控件；
- 到达后再显示企业任务点；
- 避免镜头连续快速旋转。

#### 企业选中

- 点位轻微放大；
- 对应建筑高亮；
- 右侧抽屉滑入；
- 地图中心向左偏移，为抽屉留出空间。

#### 模式切换

- 企业点位淡出；
- 热力图渐入；
- 建筑饱和度降低；
- 底部控制坞出现。

### 7.3 动效参数

```text
快速反馈：120—180ms
面板切换：240—320ms
地图图层：300—500ms
首页标题：800—1100ms
地图飞行：900—1600ms
```

### 7.4 降低动态效果

支持：

```css
@media (prefers-reduced-motion: reduce)
```

降级行为：

- 取消鼠标跟随；
- 取消背景缩放；
- 地图使用较短过渡；
- 面板直接淡入。

## 17. 三维建筑策略

### 17.1 原则

不逐栋手工建立精细模型。

采用三层策略。

### 17.2 第一层：普通建筑

占比约85%—95%。

来源：

- OpenStreetMap；
- 公开建筑轮廓；
- 地图矢量瓦片。

处理：

- 使用 `fill-extrusion` 拉伸；
- 有高度则使用高度；
- 有楼层则估算高度；
- 无数据时使用默认高度；
- 使用低饱和统一材质。

### 17.3 第二层：重点企业所在建筑

处理：

- 人工核验轮廓；
- 核验企业坐标；
- 补充楼层或高度；
- 建立企业与建筑的关联；
- 支持整栋高亮；
- 支持一栋多企业。

### 17.4 第三层：地标建筑

每镇选择1—3栋。

可选：

- Blender低模；
- SketchUp简模；
- glTF/GLB；
- 简化贴图；
- 夜间局部发光。

### 17.5 高度推断

优先级：

```text
height
→ building:levels × 默认层高
→ 官方资料
→ 现场估计
→ 建筑类型默认值
```

示例默认值：

| 类型 | 默认高度 |
|---|---:|
| 普通办公楼 | 18m |
| 厂房 | 12m |
| 展厅 | 10m |
| 科研平台 | 24m |
| 人才公寓 | 30m |

所有推断高度应标记为展示性估算。

## 25. 性能规划

### 25.1 性能目标

| 指标 | 目标 |
|---|---:|
| 首屏可交互 | 3秒内 |
| 地图初始化 | 2秒内 |
| 小镇切换 | 1.5秒内完成主要视觉切换 |
| 企业搜索响应 | 200ms内 |
| 详情面板打开 | 300ms内 |
| 桌面端帧率 | 50—60fps |
| 移动端帧率 | 30fps以上 |

### 25.2 优化策略

- 首页不加载MapLibre完整地图；
- 地图页面按小镇加载数据；
- 图片使用WebP/AVIF；
- 企业点位使用Symbol Layer；
- 启用点位聚合；
- 三维模型使用低面数；
- GLB按需加载；
- 通勤圈和热力图延迟加载；
- 大GeoJSON转换为PMTiles或矢量瓦片；
- 不在鼠标移动时高频触发Vue响应式更新；
- 动效优先使用transform和opacity。

### 25.3 性能预算

#### 首页

- 背景图片单张控制在500KB以内；
- 总资源建议小于2MB。

#### 地图页

- 初始企业数据小于300KB；
- 单镇GeoJSON小于1MB；
- 单个地标模型小于2MB；
- 单镇全部模型小于6MB。

## 34. 最终产品形态

完成后的平台应形成以下体验：

1. 用户进入一个具有电影感的全屏首页；
2. 鼠标聚光揭示地图下的产业信息；
3. 点击“开始探索”进入六镇都市圈；
4. 通过底部卡片选择小镇；
5. 镜头飞入三维园区；
6. 企业以任务点形式分布；
7. 点击企业后建筑高亮、详情抽屉展开；
8. 用户可以切换产业热力和交通区位；
9. 用户可以返回都市圈比较六镇；
10. 每项数据均能查看来源与口径。

平台的核心竞争力不是功能数量，而是：

> **通过现代、克制、沉浸式的地图交互，将六镇复杂的产业与空间信息组织成一个容易理解、值得探索、适合传播的数字体验。**
